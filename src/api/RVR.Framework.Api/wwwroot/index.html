Démarrage rapide
  • Étape 1 : Vérifiez le statut d'initialisation du système
  • Étape 2 : Si nécessaire, créez le premier utilisateur admin via POST /api/init/first-admin
  • Étape 3 : Authentifiez-vous via POST /api/auth/login pour obtenir un token JWT
  • Étape 4 : Utilisez le token pour accéder aux endpoints protégés
  • Étape 5 : Testez l'API via Swagger UI ou ReDoc
Endpoints principaux
  • Initialization : /api/init/* Public
  • Authentication : /api/auth/* Public
  • Users : /api/users/*
  • Products : /api/products/*
